# FLW-005 — FAQ accordion drive (smoke + content regression)

> Per Template 2 + §5b.2.

```
ENV:           public(dev-time) + tst + tst-demo
FLOW ID:       FLW-005
FLOW NAME (CS): Procházení FAQ kategorií a akordeon
FLOW NAME (EN): Browse FAQ categories and accordion items
ENTRY POINT:    SCR-003 (`/faq`)
EXIT POINT:     all 6 categories visited, each accordion item opened
                + closed at least once
USER ROLE:      Anonymous
```

## Step sequence (happy path) — with §5b.2 step typing

```
STEP 1 — trigger_point
  ACTION: navigate to /faq.

STEP 2 — assertion
  ACTION: H1 "Často kladené otázky" present.

STEP 3 — data_collection_point
  ACTION: capture all 6 category-button labels.

STEP 4 — assertion
  ACTION: expected category set matches (Základní info / Technické
          požadavky / Vyplňování / Bezpečnost / Výstupy / Ostatní).

STEP 5..10 — for each category:
  STEP 5a — trigger_point: click category button
  STEP 5b — assertion: associated accordion items visible
  STEP 5c — data_collection_point: capture each item's H + first 60
            chars of body
  STEP 5d — assertion: count of accordion items matches expected
            (per-env stable; per-env list filled via user template)
  STEP 5e — trigger_point: open each item (click button); assert region
            visible (ARIA expanded=true)
  STEP 5f — trigger_point: close each item; assert region hidden
            (ARIA expanded=false)
  KIND chain: trigger + assertion + data_collection + assertion +
              trigger + assertion + trigger + assertion.
```

## Failure modes (R-FAIL-1)

```
F1 — Category button click yields zero accordion items
  EXPECTED: test fails (CMS regression)

F2 — Accordion item body shorter than expected (truncated content)
  EXPECTED: warn (not fail) + capture diff for manual review
  RATIONALE: content tweaks are routine; structural failure is the
  test's primary concern, not text-stability

F3 — ARIA expanded state not toggled
  EXPECTED: test fails (accessibility regression)
```

## Env-specific divergences

DEMO may carry a school-instructor flavoured FAQ — different copy or
extra items. Per-env expected-count list filled via user template.

## Integration touchpoints

None.

## Mobile-first viewport spec

horizontal-scroll on narrow widths; assert all 6 are reachable via
swipe (or tab-to in keyboard mode).
