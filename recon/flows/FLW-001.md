## Step sequence (happy path) — with §5b.2 step typing

```
STEP 1 — trigger_point
  ACTION: navigate to landing (/)
  DETAIL: page.goto(env.BASE_URL + '/')
  KIND:   trigger_point — entry navigation; treated as deterministic
          since the dev-time public URL is reachable.

STEP 2 — assertion
  ACTION: assert page <title>
  DETAIL: expect(page).toHaveTitle(/Bouračka/)
  KIND:   assertion — confirms the landing rendered server-side.

STEP 3 — data_collection_point
  ACTION: capture H1 + primary CTA presence
  DETAIL: const h1 = page.locator('h1').first().innerText();
          const cta = page.getByRole('button', { name: /VYPLNIT ZÁZNAM/i }).first();
  KIND:   data_collection_point — DOM read; values stored for the next
          assertion.

STEP 4 — assertion
  ACTION: assert H1 matches expected + CTA visible + CTA href contains '/formular'
  DETAIL: expect(h1).toMatch(/Bouračka/);
          await expect(cta).toBeVisible();
          await expect(cta.locator('xpath=ancestor::a'))
            .toHaveAttribute('href', /\/formular/);
  KIND:   assertion — three checks; failure of any is a smoke fail.

STEP 5 — trigger_point
  ACTION: click primary CTA
  DETAIL: await cta.click();
  KIND:   trigger_point — user-action.

STEP 6 — control_point
  ACTION: branch on URL change
  DETAIL: if URL ends with '/formular' → continue; else → fail (the
          CTA may have led to /formular?utm_source=landing or
          /formular — both acceptable).
  KIND:   control_point — branch decision in the flow.

STEP 7 — data_collection_point
  ACTION: capture wizard-gateway H1 ("Stala se vám dopravní nehoda? To zvládnete.")
  DETAIL: const gatewayH1 = page.locator('h1').first().innerText();
  KIND:   data_collection_point.

STEP 8 — assertion
  ACTION: assert gateway H1 matches; assert reCAPTCHA badge present;
          assert primary CTA "Vyplnit záznam" visible
  DETAIL: expect(gatewayH1).toMatch(/dopravní nehoda/);
          await expect(page.locator('[data-recaptcha], .grecaptcha-badge'))
            .toBeAttached();
          await expect(page.getByRole('button', { name: /Vyplnit záznam/i }))
            .toBeVisible();
  KIND:   assertion — three checks.
```

## Failure modes (R-FAIL-1 obligation)

```
FAILURE F1 — landing 5xx
  TRIGGER: navigation returns 500/502/503/504
  EXPECTED: test fails fast with verdict 'fail';
            evidence: response.status() captured, screenshot taken
  KIND mapping: trigger_point (nav) + assertion (status check)

FAILURE F2 — primary CTA missing
  TRIGGER: H1 + page render, but no element matches the CTA selector
  EXPECTED: test fails with verdict 'fail';
            evidence: full-page screenshot + DOM snippet of the visible
            header for diff diagnosis
  KIND mapping: data_collection_point (locator) + assertion (visible)

FAILURE F3 — CTA visible but href != /formular
  TRIGGER: regression where the CTA was rebound to a different page
  EXPECTED: test fails; evidence: href value captured
  KIND mapping: data_collection_point + assertion

FAILURE F4 — mobile-only: CTA touch-target < 44×44 px
  TRIGGER: viewport is set to 320 / 375 / 414; bounding-box of the CTA
           wraps below the WCAG 2.2 AA target-size minimum
  EXPECTED: test fails with verdict 'fail' (mobile-only regression)
  KIND mapping: data_collection_point (boundingBox) + assertion (size)
```

mobile UA}. Failure F4 fires only at the three narrow widths.
