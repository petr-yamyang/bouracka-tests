## Failure modes worth asserting

1. `POST /api/reports` rate-limited (anti-abuse).
2. `GET /api/reports/{uuid}` for nonexistent UUID — 404 envelope.
3. State-machine illegal transitions — server should return 4xx
   (e.g. trying to sign before verifying).
4. Concurrency — two browsers same UUID — last-write-wins?
