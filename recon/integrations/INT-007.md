## Behaviour

- Bundle pre-loaded on rozcestník (probably for the D13
  datum-čas-místo screen which shows a map picker).
- **Locale anomaly:** URL path forces `intl/en_gb/` — the page UI is
  in Czech but Maps loads in **British English**. Either:
  - bundle hard-codes `language=en_GB` parameter (suspected bug);
  - geolocation override at Google's edge (unlikely);
  - Bouračka relies on UI-level translation of map labels (unusual).
- This is a **CS-locale-violation candidate**. May surface as a
  defect in the SUPIN review pass.

## Failure modes worth asserting

1. Maps API **locale mismatch** (current state — `en_gb` instead
   of `cs_CZ`): file as functional defect; flag for ČKP product side.
2. Maps API **quota exceeded** (Google quota): does the SPA show a
   degraded experience or a hard failure?
3. Maps API **blocked** (CSP violation, ad-blocker): graceful
   fallback to manual address input?

## Privacy / GDPR posture

- Google Maps **transmits user IP + viewport** to Google on each load;
  may need a row in the GDPR / cookie-consent narrative.
- Currently no cookie-consent banner observed — needs explicit check
  of CSP/cookie posture in next walk.
