WHAT YOU'RE INSTALLING
----------------------
Bouracka UI is a small local web app that lets you run the Bouracka test suite
against demo.bouracka.cz (or tst.bouracka.cz, uat.bouracka.cz, www.bouracka.cz)
from a single screen instead of remembering separate CLI commands per framework
(Cypress / Playwright / Selenium). It also displays results and lets you file
bugs back into the shared Excel workbook.

It runs ENTIRELY on YOUR laptop. No data leaves the machine (except the HTTP
requests the test scripts already make to bouracka.cz).


PREREQUISITES
-------------
You need ONE of the following Python interpreters available on PATH:
  - Python 3.10 (recommended — matches the wheel we built)
  - Python 3.11 or 3.12 (also fine; pure-Python wheel)

To check, open PowerShell and run:

    python --version

If you see "Python 3.10.x" or higher, you're good. If you get "command not
found" or a much older version, install Python 3.10 from python.org first
(check "Add Python to PATH" during installation).

OPTIONAL prerequisites — only needed if you actually want the test runs to
fire real browser automation rather than mock-mode demos:
  - Node.js 18+ (for Cypress + Playwright)
  - Cypress 13+ (npm install cypress --save-dev inside bouracka-tests/)
  - Playwright (npm install @playwright/test --save-dev; npx playwright install)

If npx is not on PATH the dispatcher falls back to MOCK MODE — the UI still
works end-to-end (you can drive it, see fake results, file bugs), it just
won't actually contact bouracka.cz.


INSTALL — FIRST TIME ON A FRESH LAPTOP
--------------------------------------

1. Unzip this package to a folder you'll keep. Recommended:

       C:\bouracka-ui

2. Open PowerShell, navigate to that folder, and create a virtual environment:

       cd C:\bouracka-ui
       python -m venv .venv
       .\.venv\Scripts\Activate.ps1

   You should now see "(.venv)" prepended to your prompt.

   If PowerShell refuses to run Activate.ps1 with "execution of scripts is
   disabled," run this ONCE as the current user (no admin rights needed):

       Set-ExecutionPolicy -Scope CurrentUser -ExecutionPolicy RemoteSigned

   ...and then re-run Activate.ps1.

3. Install the bouracka-ui wheel:

       pip install --upgrade pip
       pip install bouracka_ui-0.1.0-py3-none-any.whl

   This pulls down FastAPI, uvicorn, openpyxl, etc. (~30 MB total).

4. Smoke-check the install:

       bouracka-ui --help

   You should see the usage banner. If yes — you're installed.

5. Verify the test workbook is present alongside the install:

       ls BOURACKA-TESTPLAN-*.xlsx

   You should see "BOURACKA-TESTPLAN-v0.4.2.xlsx" (or whichever version
   ships with this package).


RUN IT
------

       bouracka-ui

This starts the local server on http://127.0.0.1:8424/ and (by default)
opens your default browser to the Run page automatically.

To stop the server: Ctrl+C in the PowerShell window. Close the browser tab
when you're done.


FIRST RUN — SANITY CHECK (TC-CP-A1-MAIN-DEMO)
---------------------------------------------

1. With bouracka-ui running, open http://127.0.0.1:8424/ in your browser.
2. On the "Run tests" page:
   - Environment: leave it on "ENV-DMO — Demo (demo.bouracka.cz)"
   - Framework: pick "cypress only" if you have Cypress installed,
                else leave on "all" — mock mode will produce fake results
3. Scroll down, tick the TC named "TC-CP-A1-MAIN-DEMO" (it's near the top).
4. Click the "Run selected" button.
5. The Live log card appears below. Watch the lines as they stream.
6. After ~30-60 s (real mode) or ~3 s (mock mode), you'll be auto-navigated
   to the /results/<run_id> page showing the verdict matrix.

WHILE THE RUN IS IN FLIGHT, the results page will show a "running" pill +
polling indicator. This is normal — it auto-refreshes every 2 s until the
envelope is written. You can navigate away (Runs / Bugs / About) and come
back via the Runs page later; the run continues in the background.


KNOWN ISSUES + FIXES
--------------------

→ If you see "Run not found: run-...." for more than ~90 s:
  The dispatcher likely failed silently (e.g. Cypress not installed but
  framework was set to "cypress only"). Check the log tail on the same page,
  or use the /about page to see which tools are detected on PATH.

→ If "bouracka-ui" startup hits "[Errno 10048] error while attempting to
  bind on address ('127.0.0.1', 8424)":
  Another bouracka-ui process is still running. Use kill-stragglers.ps1
  (bundled in this package) — see TROUBLESHOOTING.md for the exact recipe.

→ If pip install fails with "Access Denied" on websockets/speedups.cp310...
  or httptools/parser.cp310...":
  Same root cause — a stale bouracka-ui process is holding the .pyd file.
  Run kill-stragglers.ps1, then retry pip install.

→ Anything else: open TROUBLESHOOTING.md.
