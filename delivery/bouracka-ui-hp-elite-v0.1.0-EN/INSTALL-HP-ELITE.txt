WHAT YOU'RE INSTALLING
----------------------
Bouracka UI is a small local web app that lets you run the Bouracka test suite
against demo.bouracka.cz (or tst.bouracka.cz, uat.bouracka.cz, www.bouracka.cz)
from a single screen instead of remembering separate CLI commands per framework
(Cypress / Playwright / Selenium). It also displays results and lets you file
bugs back into the shared Excel workbook.

It runs ENTIRELY on YOUR laptop. No data leaves the machine.


PRE-FLIGHT CHECK — Python version + matching ZIP
------------------------------------------------
The ZIP you have should have a name like:

    bouracka-ui-hp-elite-v0.1.2-EN-py312.zip
                                    ^^^^^
                                    Python 3.12 wheels inside

On the target machine, FIRST confirm what Python version you have:

    python --version

You need a MATCH between the ZIP's `-pyNNN` suffix and your local Python:

    Python 3.10.x  →  use the -py310 ZIP
    Python 3.11.x  →  use the -py311 ZIP
    Python 3.12.x  →  use the -py312 ZIP

If you have a different Python version, ask Pete to rebuild the wheelhouse
for your version (one PowerShell command on ThinkPad, ~30 seconds).

If Python is not installed:
- Install Python 3.12 from python.org. Tick "Add Python to PATH" during
  installation. (3.10 and 3.11 also work but 3.12 is recommended.)


INSTALL — FIRST TIME ON A FRESH LAPTOP (AIR-GAP MODE)
------------------------------------------------------

1. Unzip the package to a folder you'll keep. Recommended:

       C:\TestAutomationSite\

   (or any path you prefer — the bouracka-ui auto-detects the location).

2. Confirm the wheelhouse subfolder exists with ~25-30 wheels in it:

       cd C:\TestAutomationSite
       Get-ChildItem .\wheelhouse\*.whl | Measure-Object | Select-Object Count

3. Create a Python virtual environment:

       python -m venv .venv
       .\.venv\Scripts\Activate.ps1

   You should now see "(.venv)" prepended to your prompt.

   If PowerShell refuses to run Activate.ps1 with "execution of scripts is
   disabled," run this ONCE as the current user (no admin rights needed):

       Set-ExecutionPolicy -Scope CurrentUser -ExecutionPolicy RemoteSigned

   ...and then re-run Activate.ps1.

4. Install bouracka-ui FROM THE BUNDLED WHEELHOUSE (no PyPI access needed):

       pip install --no-index `
         --find-links="C:\TestAutomationSite\wheelhouse" `
         "C:\TestAutomationSite\bouracka_ui-0.1.2-py3-none-any.whl"

   Key flags:
     --no-index           tells pip to NEVER consult PyPI
     --find-links=<path>  points to the local wheelhouse folder
     <absolute path>      avoid relative paths; pip is finicky with `.\` on Windows

   Expected output: ~28 packages "Successfully installed" — including
   bouracka-ui-0.1.2 + fastapi + uvicorn + openpyxl + sse-starlette +
   python-multipart + pytest + pytest-json-report + transitive deps.

5. Smoke-check the install:

       bouracka-ui --help

   You should see the usage banner. If yes — you're installed.

6. Verify the workbook is present alongside the install:

       Get-ChildItem .\BOURACKA-TESTPLAN-*.xlsx

   You should see "BOURACKA-TESTPLAN-v0.4.3.xlsx" (or whichever version
   ships with this package).


RUN IT
------

       bouracka-ui --no-browser

This starts the local server on http://127.0.0.1:8424/. Open that URL in your
browser. The `--no-browser` flag skips the auto-open (useful if you want to
run the server in one window and use the browser separately).

To stop the server: Ctrl+C in the PowerShell window. Close the browser tab
when you're done.

If port 8424 is already in use (because a previous bouracka-ui didn't shut
down cleanly), run the bundled `kill-stragglers.ps1` first — see TROUBLESHOOTING.md §1.


FIRST RUN — SANITY CHECK (TC-CP-A1-MAIN-DEMO, MOCK MODE)
--------------------------------------------------------
Real-mode dispatch needs Cypress / Playwright / pytest+Selenium installed
separately (typically via the automation-v0.5.4 package). For a quick first
verification that the UI plumbing works, use mock mode:

1. With the venv still active, in a NEW PowerShell window:

       $env:BOURACKA_UI_DISPATCH_MODE = "mock"
       cd C:\TestAutomationSite
       .\.venv\Scripts\Activate.ps1
       bouracka-ui --no-browser

2. Open http://127.0.0.1:8424/ in your browser.

3. On the "Run tests" page:
   - Environment: leave it on "ENV-DMO — Demo (demo.bouracka.cz)"
   - Framework: leave on "all" (mock mode handles all frameworks)

4. Scroll down, tick "TC-CP-A1-MAIN-DEMO". Click "Run selected".

5. After ~3 s the verdict matrix appears with synthetic results. UI plumbing
   confirmed end-to-end.

6. Stop the server with Ctrl+C. Remove the mock-mode override when you're
   ready for real dispatch:

       Remove-Item Env:BOURACKA_UI_DISPATCH_MODE


REAL-MODE DISPATCH — pair with automation-v0.5.4
-------------------------------------------------
For real Cypress / Playwright / pytest+Selenium runs against bouracka.cz,
you need the automation suite installed alongside this UI:

   1. Unpack `bouracka-automation-v0.5.4.zip` (paired delivery) — it contains
      `bouracka-tests-v0.5.4.zip` inside; unpack THAT too.
   2. The result is a `bouracka-tests/` tree with `tools/`, `cypress/`,
      `playwright/`, `selenium/`, fixtures, etc.
   3. Run `bouracka-ui --no-browser` from inside that bouracka-tests tree
      (the REPO_ROOT auto-detect walks up from CWD looking for
      `tools/consolidate_results.py` — present in automation-v0.5.4).
   4. If detection fails, override with:
         $env:BOURACKA_UI_REPO_ROOT = "C:\path\to\bouracka-tests"

Cypress and Playwright additionally need Node.js 18+ installed and:
   npm install                       (from the automation-v0.5.4 root)
   npx playwright install            (downloads playwright browsers)


KNOWN ISSUES + FIXES
--------------------

→ pip install fails with "getaddrinfo failed" / "No matching distribution":
  This means pip tried to reach PyPI. Make sure you used `--no-index`. The
  air-gap install MUST use --no-index --find-links to work without internet.
  Re-run step 4 with both flags.

→ pip install fails with "Could not find a version that satisfies the
  requirement <something>; extra == 'standard'" or platform-tagged wheels
  not found:
  Python version mismatch. The ZIP's `-pyNNN` suffix doesn't match your local
  Python. Confirm `python --version` and use the matching ZIP variant.

→ "Run not found: run-...." persists for >90 s:
  Dispatcher likely failed silently. Check the log on the results page
  (now shows 4 candidate causes) or use the /about page for tool availability.

→ "[Errno 10048] error while attempting to bind on address ('127.0.0.1', 8424)":
  Another bouracka-ui is still running. Use kill-stragglers.ps1 (bundled).

→ Anything else: open TROUBLESHOOTING.md, then DIAGNOSTICS-PLAYBOOK.md for
  deeper diagnosis.
