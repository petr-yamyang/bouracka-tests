CO JE NOVÉHO V v0.1.2
---------------------
- Air-gap installer: balíček nyní obsahuje složku `wheelhouse/` se všemi
  Python závislostmi jako předem-stažené wheely. ŽÁDNÝ PyPI přístup
  na cílovém stroji není potřeba. Kritické pro SUPIN HP Elite (žádný
  externí síťový egress).
- Python-version-tagged: název ZIPu obsahuje `-py312`, `-py311` nebo
  `-py310` podle toho, pro kterou Python ABI byly wheely vyrobeny.
- Všechny BUG opravy z v0.1.0: BUG-BUI-001 (NTFS filename safety),
  BUG-BUI-002 (in-flight polling), BUG-BUI-003 (přejmenování karty),
  BUG-BUI-004 (REPO_ROOT auto-detekce + Back tlačítko + multi-cause
  zpráva + pytest-json-report runtime dependency).


CO INSTALUJETE
--------------
Bouracka UI je malá lokální webová aplikace, která umožňuje spouštět
Bouračku testovací sadu proti demo.bouracka.cz (nebo tst.bouracka.cz,
uat.bouracka.cz, www.bouracka.cz) z jediné obrazovky místo zapamatovávání
si oddělených CLI příkazů pro jednotlivé frameworky (Cypress / Playwright
/ Selenium). Také zobrazuje výsledky a umožňuje zakládat bugy zpět do
sdíleného Excel workbooku.

Běží KOMPLETNĚ na VAŠEM notebooku. Žádná data neopouštějí stroj.


PRE-FLIGHT KONTROLA — Python verze + odpovídající ZIP
------------------------------------------------------
ZIP, který máte, by měl mít název typu:

    bouracka-ui-hp-elite-v0.1.2-CS-py312.zip
                                    ^^^^^
                                    Python 3.12 wheely uvnitř

Na cílovém stroji NEJDŘÍV ověřte, jakou verzi Pythonu máte:

    python --version

Potřebujete SHODU mezi sufixem ZIPu `-pyNNN` a vaším lokálním Pythonem:

    Python 3.10.x  →  použijte ZIP -py310
    Python 3.11.x  →  použijte ZIP -py311
    Python 3.12.x  →  použijte ZIP -py312

Pokud máte jinou verzi Pythonu, požádejte Peta, aby přestavěl wheelhouse
pro vaši verzi (jeden PowerShell příkaz na ThinkPadu, ~30 sekund).

Pokud Python není nainstalovaný:
- Nainstalujte Python 3.12 z python.org. Zaškrtněte „Add Python to PATH"
  během instalace. (3.10 a 3.11 také fungují, ale 3.12 je doporučená.)


INSTALACE — POPRVÉ NA ČISTÉM NOTEBOOKU (AIR-GAP REŽIM)
-------------------------------------------------------

1. Rozbalte balíček do složky, kterou si udržíte. Doporučeno:

       C:\TestAutomationSite\

   (nebo jakákoli vámi preferovaná cesta — bouracka-ui auto-detekuje
   umístění).

2. Ověřte, že wheelhouse podsložka existuje a obsahuje ~25-30 wheelů:

       cd C:\TestAutomationSite
       Get-ChildItem .\wheelhouse\*.whl | Measure-Object | Select-Object Count

3. Vytvořte Python virtual environment:

       python -m venv .venv
       .\.venv\Scripts\Activate.ps1

   Teď byste měli před promptem vidět „(.venv)".

   Pokud PowerShell odmítne spustit Activate.ps1 s „execution of scripts
   is disabled", spusťte JEDNOU jako aktuální uživatel (admin práva
   nejsou potřeba):

       Set-ExecutionPolicy -Scope CurrentUser -ExecutionPolicy RemoteSigned

   ...a pak Activate.ps1 znovu.

4. Nainstalujte bouracka-ui Z PŘIBALENÉHO WHEELHOUSE (žádný PyPI přístup
   potřeba):

       pip install --no-index `
         --find-links="C:\TestAutomationSite\wheelhouse" `
         "C:\TestAutomationSite\bouracka_ui-0.1.2-py3-none-any.whl"

   Klíčové parametry:
     --no-index           říká pipu, aby NIKDY nekontaktoval PyPI
     --find-links=<cesta> ukazuje na lokální wheelhouse složku
     <absolutní cesta>    vyhněte se relativním cestám; pip je na Windows
                          citlivý na `.\`

   Očekávaný výstup: ~28 balíčků „Successfully installed" — včetně
   bouracka-ui-0.1.2 + fastapi + uvicorn + openpyxl + sse-starlette +
   python-multipart + pytest + pytest-json-report + tranzitivní deps.

5. Smoke kontrola instalace:

       bouracka-ui --help

   Měli byste vidět banner s nápovědou. Pokud ano — máte nainstalováno.

6. Ověřte, že workbook je přítomný vedle instalace:

       Get-ChildItem .\BOURACKA-TESTPLAN-*.xlsx

   Měli byste vidět „BOURACKA-TESTPLAN-v0.4.3.xlsx" (nebo verzi
   dodanou s tímto balíčkem).


SPUŠTĚNÍ
--------

       bouracka-ui --no-browser

To spustí lokální server na http://127.0.0.1:8424/. Otevřete tuto URL ve
svém prohlížeči. Parametr `--no-browser` přeskočí auto-otevření (užitečné,
pokud chcete spustit server v jednom okně a používat prohlížeč zvlášť).

Zastavení serveru: Ctrl+C v okně PowerShellu. Záložku v prohlížeči zavřete,
až s tím skončíte.

Pokud je port 8424 už používán (protože předchozí bouracka-ui se neukončil
čistě), spusťte nejdřív přibalený `kill-stragglers.ps1` — viz
TROUBLESHOOTING-CS.md §1.


PRVNÍ BĚH — SMOKE TEST (TC-CP-A1-MAIN-DEMO, MOCK MODE)
-------------------------------------------------------
Real-mode dispatch potřebuje Cypress / Playwright / pytest+Selenium
nainstalované zvlášť (typicky přes balíček automation-v0.5.4). Pro
rychlé první ověření, že UI plumbing funguje, použijte mock mode:

1. S aktivovaným venv, v NOVÉM okně PowerShellu:

       $env:BOURACKA_UI_DISPATCH_MODE = "mock"
       cd C:\TestAutomationSite
       .\.venv\Scripts\Activate.ps1
       bouracka-ui --no-browser

2. Otevřete http://127.0.0.1:8424/ v prohlížeči.

3. Na stránce „Run tests":
   - Environment: nechte na „ENV-DMO — Demo (demo.bouracka.cz)"
   - Framework: nechte na „all" (mock mode handles all frameworks)

4. Sjeďte níž, zaškrtněte „TC-CP-A1-MAIN-DEMO". Klikněte na „Run selected".

5. Po ~3 s se objeví verdiktová matice se syntetickými výsledky. UI
   plumbing potvrzen end-to-end.

6. Zastavte server Ctrl+C. Odstraňte mock-mode override, až budete
   připraveni pro real dispatch:

       Remove-Item Env:BOURACKA_UI_DISPATCH_MODE


REAL-MODE DISPATCH — spárovat s automation-v0.5.4
--------------------------------------------------
Pro reálné Cypress / Playwright / pytest+Selenium běhy proti
bouracka.cz potřebujete automation suite nainstalovaný vedle tohoto UI:

   1. Rozbalte `bouracka-automation-v0.5.4.zip` (spárovaná dodávka) — obsahuje
      `bouracka-tests-v0.5.4.zip` uvnitř; rozbalte TEN také.
   2. Výsledek je strom `bouracka-tests/` s `tools/`, `cypress/`,
      `playwright/`, `selenium/`, fixtures atd.
   3. Spusťte `bouracka-ui --no-browser` zevnitř toho bouracka-tests stromu
      (REPO_ROOT auto-detekce jde nahoru z CWD a hledá
      `tools/consolidate_results.py` — přítomné v automation-v0.5.4).
   4. Pokud detekce selže, override pomocí:
         $env:BOURACKA_UI_REPO_ROOT = "C:\cesta\k\bouracka-tests"

Cypress a Playwright navíc potřebují nainstalovaný Node.js 18+ a:
   npm install                       (z automation-v0.5.4 kořene)
   npx playwright install            (stáhne playwright prohlížeče)


ZNÁMÉ PROBLÉMY + ŘEŠENÍ
------------------------

→ pip install selže s „getaddrinfo failed" / „No matching distribution":
  To znamená, že pip zkusil dosáhnout PyPI. Ujistěte se, že jste použili
  `--no-index`. Air-gap instalace MUSÍ použít --no-index --find-links pro
  fungování bez internetu. Spusťte znovu krok 4 s oběma parametry.

→ pip install selže s „Could not find a version that satisfies the
  requirement <něco>; extra == 'standard'" nebo platform-tagged wheely
  nenalezeny:
  Nesoulad Python verzí. Sufix ZIPu `-pyNNN` neodpovídá vašemu lokálnímu
  Pythonu. Ověřte `python --version` a použijte odpovídající variantu ZIPu.

→ „Run not found: run-...." přetrvává déle než 90 s:
  Dispatcher pravděpodobně tiše selhal. Zkontrolujte log na výsledkové
  stránce (nyní ukazuje 4 kandidátní příčiny) nebo použijte /about pro
  dostupnost nástrojů.

→ „[Errno 10048] error while attempting to bind on address ('127.0.0.1', 8424)":
  Jiný bouracka-ui stále běží. Použijte kill-stragglers.ps1 (přibalený).

→ Cokoli jiného: otevřete TROUBLESHOOTING-CS.md, pak DIAGNOSTICS-PLAYBOOK-CS.md
  pro hlubší diagnostiku.
